## ADR 템플릿

새로운 ADR을 작성할 때 다음 템플릿을 사용하세요:

```markdown
# ADR-XXXX: [제목]

**상태**: 제안됨 | 승인됨 | 폐기됨 | 대체됨
**날짜**: YYYY-MM-DD
**결정자**: [팀/역할]

## 배경
[결정이 필요한 문제 또는 상황을 설명하는 2-4문장]

## 결정
[무엇을 결정했고 어떻게 작동하는지 설명하는 3-5문장]

## 결과

**긍정적**:
- [이점 1]
- [이점 2]
- [이점 3]

**부정적**:
- [트레이드오프 1]
- [트레이드오프 2]

## 구현
[해당되는 경우 코드 예제 또는 설정, 5-15줄]

## 고려된 대안
- **[대안 1]**: [거부된 이유]
- **[대안 2]**: [거부된 이유]

## 관련
- [ADR-XXXX]: [관계 설명]
```

## 가이드라인

1. **ADR을 짧게 유지**: ADR당 30-40줄 목표
2. **"왜"에 집중**: 단순히 무엇을 결정했는지가 아닌 이유 설명
3. **대안 문서화**: 고려된 옵션 표시
4. **상태 업데이트**: 해당되는 경우 폐기됨 또는 대체됨으로 표시
5. **관련 ADR 링크**: 결정 지식 그래프 구축

## 새 ADR 추가 방법

1. 다음 ADR 번호 결정 (현재 최대: 0010)
2. 파일 생성: `adr-XXXX-short-title.md`
3. 위 템플릿 사용
4. 이 인덱스 파일에 항목 추가
5. 해당되는 경우 관련 ADR에서 링크
